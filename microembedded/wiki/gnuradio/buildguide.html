GNU Radio 创建指导

创建和安装GNU Radio，你要么下载一个 release tarball  要么用 git 客户端软件去 git 库去查看。有些操作系统也许有二进制的安装包；详情请查看以下的操作系统详细指导。

查看最新的稳定发行版本 tarball 格式的代码，请到：  

  • http://gnuradio.org/releases/gnuradio

查看最新的开发版本，请键入如下命令行：

$ git clone http://gnuradio.org/git/gnuradio.git

一般而言，如果是开发 GNU Radio 的应用程序，它需基于 GNU Radio 构件的稳定的特性和 API。建议你使用 3.3 稳定发行版（stable release）。 

To get a handle on what's going on, clone the repository (if you haven't already), then run "qgit" or one of the other git viewers on it. It will show you all of the branching and merging, diffs, etc.

首先，确保系统满足顶层 README 文件描述的关联性。

http://gnuradio.org/redmine/repositories/changes/gnuradio/README.

大多数 GNU/Linux 系统是相互关联性应用包。或许还需要通过网络或者 CD/DVD 来安装。参考下面的特定的操作系统详述指导。

编译需 5 个步骤，从指令 cd 进入 gnuradio 目录开始，用如下命令完成。

$ ./bootstrap # 如果是从 tarball 编译的话请不要执行此步骤.
$ ./configure
$ make
$ make check
$ sudo make install

这将执行所有的配置检查和构建选择、测试、和安装所有通过（检测）的构件。更详细的编译控制，请阅读 创建配置指导（BuildConfiguration）。

创建问题指导

配置问题

配置出现问题的原因一般是一些关联性无法找到。请仔细阅读输出和 configure.log 文件。故障信息会准确地告诉你什么地方和为何会出现这些错误。阅读 README 看看需要那个包。对于有些系统和发行版本，在（下面)的操作系统详述会发现配置所需的版本号和相关提示。

有些我所需的模块没有被创建     

在配置（configure）输出结束时，你会得到一列将要被创建的模块和一列不会被创建的模块。你有可能不满意这些结果，问题可能是你没有让配置（configure）文件这样去做或者是有些关联不存在。首先到 创建配置指导 （BuildConfiguration）找答案，其次查看前一文档中描述的问题.
