MAC 上安装 GNU Radio

GNU Radio 已经被编译并成功地安装到运行任何兼容 XCode 的无论 Intel 或 PPC 构架的 Mac OS X "Tiger" 10.4 或 "Leopard" 10.5 ("OSX")系统上。尽管理论上在 OSX 10.3 上运行 GNU Radio 没有问题，但是配置好后台库和安装好各种应用可能还是颇费周折。

有两个指导安装 GNU Radio 到 OSX 的版本：
  • Jon Jacky
  • RadioWare@UND
鉴于 GNU Radio 3.2 已经发行（2009-05-24），前一个版本显得有点过时但还是很有参考价值。后一个版本已经升级到 3.2，对各种版本的 OSX 提供大多数工作的脚本。

为了能够编译和运行 GNU Radio 有些经由代码安装的后台库和应用是必须的。籍借 Fink,  MacPorts, 和/或 从代码/程序段（from source / scratch） 便可得到这些。MacPorts 似乎偏爱最新的版本，这其实好坏参半。Fink 和 MacPorts 两者提供数以千计的库和应用。所以强烈建议使用这些，而不要从代码/程序段来构建。 MacPorts 将很快提供一个用于 GNU Radio 3.2 的 meta 安装版（meta-installer）："sudo port install gnuradio ursp" 将会安装所有的能够编译的 GNU Radio 的模块包括 USRP。

备注：
  • 确保 SDCC 的版本至少是 2.8.0；前面的版本的 Intel-Macs 具有实时故障。
  • GNU Radio 发行版 3.2 需要 Python 2.5 或 2.6；版本 2.4 和更早的版本的问题在 2.5 已经解决。
  • PPC-Macs 的内部硬件的设计没有高速的 USB2.0，所以没有 PPC-Mac 使用 USRP 和 USB 硬件在  32 Mega-Bytes/second (MBps) 的测试；使用 PCCard/PCMCIA- 或  USB 2.0 的 PCI 卡可以接近 MBps。Intel-Macs 的内部硬件对处理 USB2.0 进行很好的设计，即使 20" iMac 也可以达到 32 MBps。
